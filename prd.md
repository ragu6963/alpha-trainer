# One-pager: AI 러닝 코치

## 1. TL;DR
Strava API와 연동하여 러닝 데이터를 자동으로 수집하고, LLM 기반 AI 코치가 개인화된 훈련 계획을 제공하는 웹 애플리케이션입니다. 사용자가 직접 본인의 LLM API 키(OpenAI, Gemini, Claude)를 등록하여 사용하므로 서버 측 AI 비용이 없습니다. 1인 개발자가 AI 바이브 코딩으로 빠르게 구축하며, 초보 러너들이 체계적이고 안전하게 실력을 향상시킬 수 있도록 돕습니다.

## 2. Goals

### Product Goals
* Strava 사용자 중 초보 러너를 위한 AI 코칭 서비스 시장 검증
* 1인 개발 + AI 코딩으로 MVP를 빠르게 출시하여 피드백 루프 확보
* 사용자가 자체 LLM API 키를 사용하여 서버 운영 비용 제로 달성
* 개인 사이드 프로젝트로 운영 가능한 수준의 낮은 유지보수 비용 달성

### User Goals
* 과거 러닝 기록을 한눈에 파악하고 분석받기
* 현재 수준에 맞는 맞춤형 훈련 계획 받기
* 부상 위험 없이 안전하게 실력 향상하기
* 러닝 습관을 지속적으로 유지하기

### Non-Goals
* Strava를 대체하는 러닝 트래킹 기능 개발
* 고급 러너를 위한 경기 최적화 기능
* 소셜 기능 및 커뮤니티 구축
* 웨어러블 디바이스 직접 연동
* 네이티브 모바일 앱 개발 (웹 앱으로 충분)

## 3. User stories

**주인공: 러닝 입문 3개월차 지민 (28세, 직장인)**

* "러닝을 시작했는데, 매번 같은 코스만 뛰다 보니 발전이 없는 것 같아요. 어떻게 훈련해야 할지 모르겠어요."
* "유튜브 영상을 봐도 내 수준에 맞는 건지 확신이 안 서요. 내 기록을 보고 조언해줬으면 좋겠어요."
* "무리하게 뛰었다가 무릎이 아팠던 적이 있어요. 안전하게 거리를 늘리고 싶어요."
* "Strava에 기록은 쌓이는데, 이걸 어떻게 활용해야 할지 모르겠어요."

**Jobs-to-be-done:**
* 내 러닝 데이터를 분석하여 현재 수준 파악하기
* 다음 훈련에서 무엇을 어떻게 해야 할지 구체적인 가이드 받기
* 점진적이고 안전한 발전 경로 확인하기

## 4. Design & Tech stack

> 디자인 가이드: [design-guide.md](design-guide.md)

### 개발 원칙
* **AI 바이브 코딩**: Claude Code 등 AI 도구를 적극 활용하여 빠른 개발
* **1인 개발 최적화**: 관리 포인트 최소화, 매니지드 서비스 우선 선택
* **풀스택 TypeScript**: 프론트/백 동일 언어로 컨텍스트 스위칭 최소화

> 상세 기술 스택 및 버전: [tech-stack.md](tech-stack.md)

### 선택 스택
| 레이어 | 기술 | 선택 이유 |
|--------|------|-----------|
| 프레임워크 | **Next.js (App Router)** | 풀스택 단일 프로젝트, Vercel 배포 용이 |
| 언어 | **TypeScript** | 타입 안전성, AI 코딩 도구와의 호환성 |
| 스타일링 | **Tailwind CSS + shadcn/ui** | 빠른 UI 구축, 일관된 디자인 시스템 |
| DB | **Supabase (PostgreSQL)** | 인증/DB/스토리지 통합, 무료 티어 |
| ORM | **Prisma** | 타입 안전 DB 접근, 마이그레이션 관리 |
| 인증 | **Strava OAuth → Supabase Auth (custom token)** | Strava OAuth 완료 후 Supabase에 custom access token 발급하여 세션 관리 |
| AI/LLM | **Vercel AI SDK + Gemini / OpenAI / Claude** (사용자 API 키) | 프로바이더 추상화 및 스트리밍을 AI SDK로 통합 처리. MVP: Gemini 3.1 Flash, P1: GPT-5.4 mini, Claude Sonnet 4.6 |
| 배포 | **Vercel** | Next.js 최적 배포, 무료 티어 |
| 차트 | **Recharts** | React 네이티브 차트, 가벼움 |

## 5. Functional requirements

> 상세 기능 요구서: [requirements.md](requirements.md)

### P0 (Must-have) — MVP
* 인증: Strava OAuth 연동 → Supabase Auth custom token 발급으로 세션 관리 (다수 사용자 지원)
* 데이터: Strava API 러닝 기록 수집 및 DB 저장, 수동 동기화
* LLM API 키 관리: MVP에서는 Gemini 1개 프로바이더 우선 지원, 이후 OpenAI/Claude 확장
* AI 코치: Vercel AI SDK 기반 채팅 인터페이스, 스트리밍 응답, 훈련 계획 생성, 자동 데이터 범위 (최근 4주/20회 + 전체 요약)
* 대시보드: 최근 활동 목록, 주간 요약 통계, 활동 상세 보기
* 데이터 관리: 사용자 데이터 삭제 (계정 탈퇴) 기능

### P1 (Should-have)
* 추가 LLM 프로바이더 지원 (OpenAI, Claude)
* 데이터 시각화 (주간/월간 차트), 대화 히스토리, Strava Webhook 자동 동기화

### P2 (Nice-to-have)
* 목표 설정, 이메일 알림, 진행 상황 피드백

## 6. User experience

### 핵심 사용자 여정

**첫 방문 및 온보딩**
* 랜딩 페이지 접속 → "Strava로 시작하기" 버튼 클릭
* Strava OAuth 인증 화면 → 권한 승인
* 데이터 수집 진행 중 로딩 화면 (진행 상태 표시)
* LLM API 키 등록 안내 → 설정 페이지에서 API 키 입력
* 온보딩 완료 → 대시보드 진입

**일상적인 사용 흐름**
* 대시보드에서 최근 러닝 기록 요약 확인
* "AI 코치에게 물어보기" 버튼 클릭
* 채팅 인터페이스에서 질문 입력 (예: "다음 훈련 어떻게 하면 될까요?")
* AI가 기록 분석 후 구체적인 훈련 계획 제시
* 추천 받은 계획 저장 또는 바로 Strava 앱으로 이동

**데이터 동기화**
* Strava에서 새로운 러닝 활동 완료
* 앱 재방문 시 "새로운 활동 발견" 알림
* "동기화" 버튼 클릭 → 최신 데이터 업데이트

### 페이지 구성
| 페이지 | 경로 | 설명 |
|--------|------|------|
| 랜딩 | `/` | 서비스 소개 + Strava 로그인 CTA |
| 대시보드 | `/dashboard` | 러닝 기록 요약, 최근 활동, 주간 통계 |
| AI 코치 | `/coach` | 채팅 인터페이스, 대화 히스토리 |
| 활동 상세 | `/activities/[id]` | 개별 러닝 기록 상세 보기 |
| 설정 | `/settings` | Strava 연동 관리, LLM API 키 관리, 프로필 설정 |

### Edge cases & UI notes

* **데이터 없음**: Strava 계정에 러닝 기록이 없는 경우 → 빈 상태 안내 메시지
* **API 연결 오류**: Strava API 통신 실패 시 → 재시도 버튼 + toast 알림
* **토큰 만료**: OAuth 토큰 만료 시 → 자동 리프레시, 실패 시 재인증 안내
* **대용량 데이터**: 수년치 기록 → 최근 6개월 우선 로드, 나머지 백그라운드 처리
* **LLM 응답 지연**: AI 코치 응답 생성 중 → 스트리밍 응답 + 타이핑 인디케이터
* **API 키 미등록**: LLM API 키 없이 AI 코치 접근 시 → 설정 페이지로 안내
* **API 키 오류**: 잘못된 키 또는 잔액 부족 → 구체적 에러 메시지 표시 및 키 재확인 안내
* **Strava Rate limit 초과**: 15분당 100회 초과 시 → 남은 대기 시간 안내, 자동 재시도 큐
* **부적절한 요청**: 러닝과 무관한 질문 → 시스템 프롬프트로 범위 제한

## 7. Narrative

**월요일 아침 7시, 지민의 하루**

출근 전 가벼운 러닝을 마친 지민은 사무실 책상에 앉아 커피를 마시며 웹 브라우저에서 앱을 엽니다. 어제 뛴 5km 기록이 자동으로 동기화되어 있네요.

"지난주보다 페이스가 30초 빨라졌어요!"

대시보드 상단에 뜬 작은 축하 메시지에 미소가 지어집니다. 하지만 동시에 궁금증이 생깁니다. '이제 거리를 늘려야 하나, 페이스를 더 높여야 하나?'

"AI 코치" 탭을 눌러 채팅창을 엽니다.

**지민**: "이번 주 어떻게 훈련하면 좋을까요?"

몇 초 후, AI 코치가 스트리밍으로 답변합니다.

**AI 코치**: "지난 2주간 기록을 분석했어요. 총 4회 러닝, 평균 4.8km를 5'40"/km 페이스로 달리셨네요. 꾸준히 잘 하고 계세요!

이번 주는 '기초 체력 다지기'에 집중하면 좋겠어요:

- 화요일: 회복 러닝 3km (편안한 페이스)
- 목요일: 5km 이지 런 (대화 가능한 속도)
- 토요일: 6km 롱런 (지난주보다 1km만 늘리기)

토요일 6km가 부담스러우면 5.5km로 시작해도 괜찮아요. 급하게 늘리는 것보다 꾸준함이 중요합니다!"

지민은 이 내용을 저장하고, 캘린더에 표시해둡니다.

**수요일 저녁**, 예정에 없던 야근으로 화요일 훈련을 건너뛰게 됐습니다.

**지민**: "화요일 못 뛰었는데 괜찮을까요?"

**AI 코치**: "전혀 문제없어요! 오히려 충분한 휴식도 훈련의 일부입니다. 목요일 5km만 잘 소화하고, 토요일 6km를 목표로 가시면 됩니다. 무리하지 마세요."

**토요일 아침**, 6km 롱런을 무사히 완료했습니다. 앱을 열자 새로운 메시지가 떠 있습니다.

"첫 6km 완주 축하드려요! 평균 페이스도 안정적으로 유지하셨네요. 다음 주는 어떻게 할까요?"

지민은 벌써 다음 주 훈련이 기대됩니다.

## 8. Success metrics

### 핵심 지표 (1인 개발 기준)
* **Strava 연동 성공률**: OAuth → 데이터 수집 완료까지의 전환율 (목표 90%+)
* **AI 코치 대화 수**: 주간 총 AI 코치 대화 세션 수
* **재방문율**: 주 1회 이상 재방문하는 사용자 비율
* **에러율**: API 호출 실패 및 서버 에러 비율 (목표 5% 미만)

### 부가 지표
* **평균 대화 길이**: 세션당 평균 메시지 수
* **훈련 계획 실행률**: AI 추천 후 실제 Strava 기록 기반 추정
* **페이지 로드 시간**: 대시보드 초기 로드 3초 이내
