# 디자인 가이드: AI 러닝 코치

> 상위 문서: [prd.md](prd.md) | 기술 스택: [tech-stack.md](tech-stack.md)
> 최종 업데이트: 2026-03-22

이 문서는 전체 UI에서 일관된 디자인을 유지하기 위한 기준입니다.
구현 시 이 가이드를 참조하여 컴포넌트, 색상, 간격, 타이포그래피 등을 통일합니다.

---

## 1. 디자인 원칙

| 원칙 | 설명 |
|------|------|
| **간결함** | 초보 러너가 한눈에 이해할 수 있는 UI. 불필요한 장식 최소화 |
| **격려적 톤** | 색상과 문구 모두 긍정적이고 따뜻한 느낌. 경고/에러도 위협적이지 않게 |
| **데이터 우선** | 숫자와 통계가 시각적으로 돋보이도록. 러닝 기록이 주인공 |
| **모바일 퍼스트** | 모바일 레이아웃을 먼저 설계하고 데스크톱으로 확장 |

## 2. 색상 시스템

shadcn/ui의 CSS 변수 기반 테마를 사용하며, `@theme` 디렉티브로 확장합니다.

### 2.1 시맨틱 컬러 (shadcn/ui 기본)

| 용도 | CSS 변수 | 설명 |
|------|----------|------|
| 배경 | `--background` | 페이지 배경 |
| 전경 | `--foreground` | 기본 텍스트 |
| 카드 | `--card`, `--card-foreground` | 카드 컴포넌트 |
| 주요 액션 | `--primary`, `--primary-foreground` | CTA 버튼, 주요 링크 |
| 보조 | `--secondary`, `--secondary-foreground` | 보조 버튼, 태그 |
| 강조 | `--accent`, `--accent-foreground` | 호버, 선택 상태 |
| 파괴적 | `--destructive`, `--destructive-foreground` | 삭제, 경고 액션 |
| 뮤트 | `--muted`, `--muted-foreground` | 비활성 텍스트, 힌트 |
| 테두리 | `--border` | 카드, 인풋 등 테두리 |

### 2.2 앱 전용 컬러 토큰

러닝 앱 특성에 맞는 추가 시맨틱 토큰을 `@theme`에 정의합니다.

```css
@theme {
  /* 러닝 데이터 시각화 */
  --color-pace: oklch(0.65 0.15 145);       /* 초록 계열 — 페이스 */
  --color-distance: oklch(0.60 0.15 250);   /* 파랑 계열 — 거리 */
  --color-heartrate: oklch(0.65 0.20 25);   /* 빨강 계열 — 심박수 */
  --color-elevation: oklch(0.65 0.10 75);   /* 앰버 계열 — 고도 */
  --color-time: oklch(0.60 0.15 290);       /* 보라 계열 — 시간 */

  /* 상태 */
  --color-success: oklch(0.70 0.15 150);    /* 성공, 개선됨 */
  --color-warning: oklch(0.75 0.15 85);     /* 주의, Rate limit */
  --color-info: oklch(0.70 0.12 240);       /* 안내, 동기화 */

  /* Strava 브랜드 (OAuth 버튼 전용) */
  --color-strava: #fc4c02;
}
```

### 2.3 다크 모드

- shadcn/ui의 `dark` 클래스 기반 다크 모드 지원
- MVP에서는 시스템 설정 자동 감지 (`prefers-color-scheme`)
- 앱 전용 컬러 토큰도 다크 모드에서 밝기/채도 조정

### 2.4 색상 사용 규칙

| 상황 | 색상 |
|------|------|
| 주요 CTA 버튼 | `primary` |
| Strava 로그인 버튼 | `--color-strava` (#fc4c02) + 흰색 텍스트 |
| 통계 카드 아이콘/수치 강조 | 데이터 컬러 (`pace`, `distance`, `heartrate` 등) |
| 차트 시리즈 | 데이터 컬러 (일관되게 거리=파랑, 페이스=초록 등) |
| 성공 토스트 | `success` |
| 에러 토스트 / 계정 삭제 | `destructive` |
| 비활성 텍스트, 타임스탬프 | `muted-foreground` |

## 3. 타이포그래피

### 3.1 폰트

| 용도 | 폰트 | 비고 |
|------|------|------|
| 본문 | **Pretendard** | 한국어 최적화 가변 폰트. `next/font/local`로 로드 |
| 숫자/데이터 | **Geist Mono** (또는 Pretendard tabular) | 러닝 통계에서 숫자 정렬을 위해 고정폭 사용 |

### 3.2 타이포 스케일

Tailwind 기본 스케일을 사용하되, 용도별로 고정합니다.

| 용도 | Tailwind 클래스 | 크기 | 비고 |
|------|-----------------|------|------|
| 페이지 제목 | `text-2xl font-bold` | 24px | 대시보드, 설정 등 페이지 헤더 |
| 섹션 제목 | `text-lg font-semibold` | 18px | 카드 그룹 제목, "이번 주 요약" 등 |
| 카드 제목 | `text-sm font-medium` | 14px | 통계 카드 라벨 ("총 거리", "러닝 횟수") |
| 통계 숫자 (대) | `text-3xl font-bold` | 30px | 통계 카드 메인 수치 ("24.5 km") |
| 통계 숫자 (소) | `text-xl font-semibold` | 20px | 활동 상세 수치 |
| 본문 | `text-sm` | 14px | 일반 텍스트, 활동 목록 항목 |
| 보조 텍스트 | `text-xs text-muted-foreground` | 12px | 타임스탬프, 힌트, 마지막 동기화 |
| AI 채팅 메시지 | `text-sm leading-relaxed` | 14px | 채팅 버블 내 텍스트, 여유로운 줄간격 |

### 3.3 숫자 표기 규칙

러닝 데이터는 통일된 포맷으로 표시합니다.

| 데이터 | 포맷 | 예시 |
|--------|------|------|
| 거리 | 소수점 2자리 + " km" | `5.23 km` |
| 페이스 | 분'초" /km | `5'42" /km` |
| 시간 (짧은) | 분:초 | `28:35` |
| 시간 (긴) | 시간h 분m | `1h 23m` |
| 심박수 | 정수 + " bpm" | `152 bpm` |
| 고도 | 정수 + " m" | `87 m` |
| 날짜 | YYYY.MM.DD (ddd) | `2026.03.20 (금)` |
| 상대 시간 | "N분 전", "N시간 전", "N일 전" | `3시간 전` |

## 4. 간격 및 레이아웃

### 4.1 간격 규칙

Tailwind의 spacing 스케일을 일관되게 사용합니다.

| 용도 | 간격 | Tailwind |
|------|------|----------|
| 페이지 외부 패딩 | 16px (모바일), 24px (데스크톱) | `px-4 lg:px-6` |
| 페이지 최대 너비 | 1280px | `max-w-7xl mx-auto` |
| 섹션 간 간격 | 24px | `space-y-6` |
| 카드 내부 패딩 | 16px (모바일), 24px (데스크톱) | `p-4 lg:p-6` |
| 카드 간 간격 | 16px | `gap-4` |
| 폼 필드 간 간격 | 16px | `space-y-4` |
| 인라인 요소 간격 | 8px | `gap-2` |
| 아이콘과 텍스트 간격 | 8px | `gap-2` |

### 4.2 브레이크포인트

| 이름 | 기준 | Tailwind 접두사 |
|------|------|-----------------|
| 모바일 | < 768px | (기본, 접두사 없음) |
| 태블릿 | 768px ~ 1023px | `md:` |
| 데스크톱 | 1024px+ | `lg:` |

### 4.3 페이지 레이아웃 구조

```
┌─────────────────────────────────────────┐
│  네비게이션 (데스크톱: 상단 바)            │
├─────────────────────────────────────────┤
│                                         │
│  max-w-7xl mx-auto px-4 lg:px-6        │
│  ┌─────────────────────────────────┐    │
│  │  페이지 콘텐츠                    │    │
│  │  (페이지별 그리드/레이아웃)         │    │
│  └─────────────────────────────────┘    │
│                                         │
├─────────────────────────────────────────┤
│  네비게이션 (모바일: 하단 탭 바)          │
└─────────────────────────────────────────┘
```

### 4.4 그리드 패턴

| 패턴 | 모바일 | 데스크톱 | 사용처 |
|------|--------|----------|--------|
| 통계 카드 | 1열 | 3열 (`grid-cols-3`) | 대시보드 주간 요약 |
| 2컬럼 | 1열 | 사이드바 + 메인 (`grid-cols-[280px_1fr]`) | AI 코치, 설정 |
| 데이터 카드 | 1열 | 2~3열 (`grid-cols-2 lg:grid-cols-3`) | 활동 상세 |
| 목록 | 풀 너비 | 풀 너비 | 활동 목록 |

## 5. 컴포넌트 스타일 가이드

shadcn/ui (new-york 스타일) 컴포넌트를 기본으로 사용하되, 앱 맥락에 맞게 조합합니다.

### 5.1 버튼

| 변형 | 용도 | 예시 |
|------|------|------|
| `default` | 주요 액션 | "동기화", "전송" |
| `secondary` | 보조 액션 | "취소", "이전" |
| `outline` | 삼차 액션 | "더보기", 필터 |
| `destructive` | 삭제/위험 | "계정 삭제" |
| `ghost` | 최소 강조 | 네비게이션 항목, 아이콘 버튼 |
| 커스텀 (Strava) | Strava OAuth | "Strava로 시작하기" — 배경 `--color-strava`, 텍스트 흰색 |

**크기 규칙:**
- 페이지 주요 CTA: `size="lg"`
- 일반 버튼: `size="default"`
- 인라인/밀집 영역: `size="sm"`
- 모바일 터치 타겟: 최소 높이 44px

### 5.2 카드

통계, 활동 항목, 설정 섹션 등에 일관되게 `<Card>` 사용.

```
┌─ Card ─────────────────────────┐
│  CardHeader                     │
│    CardTitle    [아이콘] 제목    │
│    CardDescription  보조 설명   │
│  CardContent                    │
│    주요 콘텐츠                   │
│  CardFooter (선택)              │
│    액션 버튼                     │
└─────────────────────────────────┘
```

**통계 카드 패턴:**
```
┌─────────────────┐
│  🏃 총 거리       │  ← CardTitle (text-sm, muted-foreground, 아이콘 포함)
│  24.5 km         │  ← 주 수치 (text-3xl font-bold, 데이터 컬러)
│  지난주 대비 +12% │  ← 보조 정보 (text-xs, success/destructive 컬러)
└─────────────────┘
```

### 5.3 활동 목록 항목

```
┌─ 활동 항목 ──────────────────────────────────────────┐
│  [아이콘]  활동명                        2026.03.20   │
│           5.23 km  ·  28:35  ·  5'28" /km            │
└──────────────────────────────────────────────────────┘
```

- 클릭 가능 (전체 영역 `cursor-pointer`)
- 호버: `bg-accent` 배경
- 구분선: `border-b` 또는 카드 간 `gap-2`

### 5.4 채팅 버블

```
              ┌──────────────────────┐
              │  사용자 메시지         │  ← bg-primary, primary-foreground
              └──────────────────────┘
                                        정렬: 우측

┌──────────────────────────────────┐
│  AI 코치 메시지                    │  ← bg-muted, foreground
│  마크다운 렌더링 지원               │
└──────────────────────────────────┘
  정렬: 좌측
```

- 사용자 메시지: `bg-primary text-primary-foreground rounded-2xl rounded-br-sm`
- AI 메시지: `bg-muted rounded-2xl rounded-bl-sm`
- 최대 너비: `max-w-[85%]` (모바일), `max-w-[70%]` (데스크톱)
- AI 메시지 내 마크다운: 리스트, 볼드, 코드 블록 스타일 적용

### 5.5 토스트 알림

shadcn/ui `<Toast>` (Sonner) 사용.

| 유형 | 용도 | 예시 |
|------|------|------|
| 성공 | 작업 완료 | "5개의 새로운 활동이 동기화되었습니다" |
| 에러 | 실패 | "Strava 연결에 실패했습니다. 다시 시도해주세요." |
| 경고 | 주의 | "API 요청 한도 초과. 12분 후 자동 재시도합니다." |
| 정보 | 안내 | "새로운 활동이 발견되었습니다" |

- 위치: 우측 하단 (데스크톱), 상단 중앙 (모바일)
- 자동 닫힘: 5초

### 5.6 빈 상태 (Empty State)

데이터가 없는 화면에 일관된 패턴을 적용합니다.

```
┌─────────────────────────────────┐
│                                 │
│         [일러스트/아이콘]         │  ← Lucide 아이콘 (48px, muted-foreground)
│                                 │
│        메인 메시지               │  ← text-lg font-medium
│        보조 설명                 │  ← text-sm text-muted-foreground
│                                 │
│        [CTA 버튼]               │  ← 다음 행동 유도
│                                 │
└─────────────────────────────────┘
```

| 화면 | 아이콘 | 메시지 | CTA |
|------|--------|--------|-----|
| 대시보드 (활동 없음) | `Activity` | "아직 러닝 기록이 없습니다" | "Strava 동기화하기" |
| AI 코치 (첫 대화) | `MessageSquare` | "AI 코치에게 무엇이든 물어보세요" | 추천 질문 칩 |
| AI 코치 (키 미등록) | `Key` | "AI 코치를 사용하려면 API 키가 필요합니다" | "설정에서 등록하기" |

## 7. 네비게이션

### 7.1 데스크톱: 상단 네비게이션 바

```
┌───────────────────────────────────────────────────────────┐
│  [로고] Alpha Trainer     대시보드  AI 코치  설정    [로그아웃] │

```

- 배경: `bg-background border-b`
- 현재 페이지: `text-primary font-medium`
- 비활성: `text-muted-foreground`
- 고정: `sticky top-0 z-50`

### 7.2 모바일: 하단 탭 바

```
┌───────────────────────────────────────┐

│   라벨         라벨         라벨       │
└───────────────────────────────────────┘
```

- 높이: 64px (`h-16`)
- 배경: `bg-background border-t`
- 현재 탭: `text-primary`
- 비활성: `text-muted-foreground`
- 고정: `fixed bottom-0 z-50`
- Safe area: `pb-safe` (노치 대응)

## 8. 피드백 & 마이크로인터랙션

### 8.1 로딩 상태

| 상황 | 패턴 |
|------|------|
| 페이지 전환 | 상단 프로그레스 바 (얇은 선) |
| 데이터 로딩 | 스켈레톤 UI (shadcn/ui `<Skeleton>`) |
| 동기화 중 | 프로그레스 바 + "N개 동기화 중..." 텍스트 |
| AI 응답 대기 | 타이핑 인디케이터 (점 3개 애니메이션) |
| 버튼 처리 중 | 버튼 비활성 + 스피너 아이콘 (`<Loader2 className="animate-spin" />`) |

### 8.2 트랜지션

- 페이지 전환: 없음 (SSR/RSC 기반으로 즉시 렌더)
- 토스트 등장/퇴장: `slide-in-from-bottom` / `fade-out` (Sonner 기본)
- 호버: `transition-colors duration-150`
- 사이드바 토글 (모바일): `transition-transform duration-200`

### 8.3 포커스 & 접근성

- 모든 인터랙티브 요소에 포커스 링: `focus-visible:ring-2 focus-visible:ring-ring`
- 키보드 네비게이션 지원 (Tab, Enter, Escape)
- 충분한 색상 대비 (WCAG AA 기준, 4.5:1 이상)
- 이미지/아이콘에 `aria-label` 또는 `sr-only` 텍스트

## 10. 문구 톤 가이드

### 10.1 원칙
- **친근하고 격려적**: "잘하고 계세요!", "괜찮아요, 쉬는 것도 훈련이에요"
- **구체적**: 추상적 표현 대신 숫자와 사실 기반
- **간결**: 한 문장으로 전달 가능하면 한 문장으로

### 10.2 UI 문구 예시

| 상황 | 문구 |
|------|------|
| 동기화 완료 | "5개의 새로운 활동이 동기화되었습니다" |
| 활동 없음 | "아직 러닝 기록이 없습니다. Strava에서 달린 후 동기화해보세요!" |
| API 키 검증 성공 | "API 키가 확인되었습니다" |
| API 키 검증 실패 | "API 키를 확인해주세요. 올바른 키인지 다시 확인해보세요." |
| 계정 삭제 확인 | "모든 데이터가 영구적으로 삭제됩니다. 되돌릴 수 없습니다." |
| Rate limit | "요청 한도에 도달했습니다. 12분 후 자동으로 다시 시도합니다." |
| 토큰 만료 | "Strava 연결이 만료되었습니다. 다시 연결해주세요." |
